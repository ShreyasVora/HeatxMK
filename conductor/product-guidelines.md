# Product Guidelines: HeatxMK

## Overview
These guidelines define the prose style, UX principles, and overall branding for the HeatxMK project. Adhering to these guidelines ensures a consistent and high-quality experience for both users and developers.

## Prose Style
- **Concise & Efficient:** All documentation and user-facing messages must be short, punchy, and to the point. Focus on delivering information quickly without unnecessary detail.

## UX Principles
- **Minimalist Design:** Prioritize simple, uncluttered interfaces. Every element on the screen should serve a clear purpose. Avoid unnecessary visual noise.

## Error Handling
- **Simplified User Errors:** Use simple, non-technical language for all errors. Focus on letting the user know something went wrong in a way they can understand, without overwhelming them with technical jargon.

## Branding & Aesthetics
- **Modern & Sleek:** The visual aesthetic should be modern, clean, and professional. Use a limited, high-contrast color palette and plenty of whitespace to create a sleek look and feel.
